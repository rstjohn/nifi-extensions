Description:

This processor consumes every flowfile that it receives. It is intended to be used for during debugging. In production flows, where data loss is unacceptable, this processor should not be used.

Relationships:

nowhere
this relationship never receives any data. It exists solely because NiFi does not behave properly when flows contain processors with no relationships. At the time of this writing, template backup and restore did not work properly for processor without at least one relationship.
