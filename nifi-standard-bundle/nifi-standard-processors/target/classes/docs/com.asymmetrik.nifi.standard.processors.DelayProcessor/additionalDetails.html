Description:

Imposes a configurable delay on each flowfile prior to sending it to the success relationship.

Basic Use Case:

This processor is useful for catching failed, upstream processors so that each flowfile can be fed back to its upstream processor for the purpose of retrying. In essence, this processor enables a delayed retry capability.

Relationships:

success each flowfile will wait the configured delay and then be sent to this relationship. Order is not guaranteed. That is, flowfiles entering this processor within the same millisecond may exit the processor in a different order.
